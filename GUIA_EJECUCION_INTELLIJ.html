🚀 Guía de Ejecución

Cómo poner en marcha el proyecto desde IntelliJ IDEA paso a paso.

💻 En esta computadora

1
Abrir el proyecto
Abre IntelliJ IDEA y selecciona Open. Navega hasta la carpeta back_anajuli y presiona OK.
2
Confirmar Importación de Gradle
IntelliJ detectará automáticamente que es un proyecto de Gradle. Si aparece un aviso en la esquina inferior derecha, haz clic en Load Gradle Project.
3
Ejecutar la aplicación
Localiza el archivo CatalogApplication.java en: 
src/main/java/com/ias/catalog/CatalogApplication.java
Haz clic derecho sobre el archivo y selecciona Run 'CatalogApplication.main()'.
✨ Tip: Una vez ejecutado por primera vez, podrás usar el botón verde de "Play" en la barra de herramientas superior.

🌍 Al clonar en otras computadoras

1
Clonar el repositorio
Usa el comando: 
git clone https://github.com/Superdroid7/Backend_Cursos.git
2
Importar como Proyecto Gradle
Al abrirlo en IntelliJ, asegúrate de que se importe como Gradle Project. Esto descargará automáticamente todas las dependencias de Spring Boot.
3
Verificar el JDK
Ve a File > Project Structure > Project y asegúrate de que el SDK sea Java 17.
✨ Tip: Al usar Java Toolchains, Gradle se encargará de buscar o incluso descargar el JDK 17 automáticamente si no lo tienes configurado. ¡Es magia portable!

✅ Verificación de arranque

Para confirmar que el backend está corriendo correctamente, abre tu navegador o Postman en:

http://localhost:8080/api/courses

Deberías ver un JSON con los 6 cursos (Python, Java, AWS, etc.).

🗃️ Base de Datos y Frontend

1
Conexión a la Base de Datos
El proyecto ya está configurado para conectarse a una base de datos en la nube (Neon DB) con PostgreSQL. Las credenciales se encuentran en application.properties y las dependencias de Spring Data JPA están en build.gradle. Al arrancar la aplicación, se crean y pueblan las tablas automáticamente gracias a frontend/db/schema.sql.
2
Probar la interfaz de usuario
Abre el archivo frontend/index.html haciendo doble clic para abrirlo directamente en tu navegador web. Esta interfaz está lista para consumir el backend que tienes corriendo en tu máquina (CORS ya está habilitado en el Controller).
3
Realizar consultas
Usa la barra de búsqueda o los filtros en la interfaz HTML para probar la lógica de dominio del backend. ¡Verás que los resultados se traen de manera remota desde la base de datos en Neon DB y se visualizan en la tabla!
