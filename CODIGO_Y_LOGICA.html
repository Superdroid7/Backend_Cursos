🚀 CatalogApplication.java Entry Point
¿Qué hace?

Punto de entrada de la aplicación. Llama a SpringApplication.run() que arranca el contenedor IoC de Spring, registra todos los beans y levanta el servidor embebido Tomcat en el puerto 8080.

@SpringBootApplication
public class CatalogApplication {
    public static void main(String[] args) {
        SpringApplication.run(CatalogApplication.class, args);
    }
}
🎮 Analogía
🗺️ League of Legends Es el Nexo. Si el Nexo cae, todo el juego termina. Si esta clase no arranca, no existe el servidor. Es el corazón de toda la partida. SpringApplication.run() es el momento en que aparece la pantalla de carga y se inicializa el mapa.
📚 Definición de Clase
Clase pública normal — No hereda de nada. Es la única que tiene main(String[] args). La anotación @SpringBootApplication hace el trabajo pesado de configuración automática.
🏷️ CourseCategory.java Domain
¿Qué hace?

Define las categorías válidas de un curso. Al ser un enum, los únicos valores posibles son exactamente los listados: PROGRAMMING, DESIGN, CLOUD, TESTING. Imposible ingresar un dato inválido.

public enum CourseCategory {
    PROGRAMMING, DESIGN, CLOUD, TESTING
}
🎮 Analogía
🧟 Resident Evil Es como el tipo de mutación del virus. Un zombie solo puede ser T-Virus, G-Virus, C-Virus o Plaga — no puede ser una mezcla aleatoria. El enum garantiza que ningún curso tenga una categoría inventada.
📚 Definición de Clase
enum — Tipo especial de clase en Java. No se instancia con new. Sus valores son constantes predefinidas en tiempo de compilación. Mas seguro que usar Strings para representar estados o tipos.
📘 Course.java Domain
¿Qué hace?

Entidad central del dominio. Representa un curso con sus datos. Es inmutable: todos los campos son final y no hay setters. Se construye con el patrón Builder de Lombok.

@Getter @Builder @AllArgsConstructor
public class Course {
    private final Long id;
    private final String title;
    private final String description;
    private final CourseCategory category;
    private final String imageUrl;
    private final String themes;
    private final String link;
}
🎮 Analogía
👤 Silent Hill 2 James Sunderland busca a su esposa en la niebla. James es inmutable: sus memorias y su identidad no cambian durante su viaje por el pueblo (el dominio). Nadie desde afuera puede modificar quién es. Solo puede ser "leído" — igual que nuestro objeto Course.
📚 Definición de Clase
Clase POJO inmutable con Builder — @Getter genera los getX(), @Builder genera el patrón builder, @AllArgsConstructor genera el constructor necesario para Builder + campos final. Sin setters = objeto inmutable = thread-safe.
🔌 CourseUseCase.java — Puerto de Entrada Domain
¿Qué hace?

Interfaz que define los casos de uso disponibles. El controller SOLO puede hablar con el dominio a través de esta interfaz. Garantiza que la capa web no acceda directamente al servicio.

public interface CourseUseCase {
    List<Course> findAll();
    List<Course> search(String keyword, CourseCategory category);
    Optional<Course> findById(Long id);
    Course createCourse(Course course);
    // + searchByKeyword / filterByCategory
}
🎮 Analogía
🎯 League of Legends Es el libro de habilidades de un campeón. Define CUÁLES habilidades existen (Q, W, E, R), pero no CÓMO se ejecutan internamente. El jugador (controller) solo ve el libro. El código real de la habilidad (CourseService) está encapsulado.
📚 Definición de Clase
interface — Contrato puro. Solo define firmas de métodos. No tiene implementación. Permite el principio D de SOLID (Dependency Inversion): depender de abstracciones, no de concretos.
🔌 CourseRepositoryPort.java — Puerto de Salida Domain
¿Qué hace?

El dominio define QUÉ necesita del almacenamiento sin saber si es una base de datos, un archivo, o datos en memoria. La implementación concreta (InMemoryCourseRepository) vive en infraestructura.

public interface CourseRepositoryPort {

🎮 Analogía

📤 CourseResponse.java y CourseRequest.java — DTOs Infra
¿Qué hace?

Data Transfer Objects: el JSON que el frontend recibe y envía. CourseRequest agrupa los datos para crear un curso, mientras CourseResponse es lo que se devuelve. Están anotados con @Schema para autogenerar la documentación de Swagger.

@Getter @Builder
public class CourseResponse {
    private Long id;
    private String title, description, category, imageUrl;

    public static CourseResponse fromDomain(Course c) {
        return CourseResponse.builder()
            .id(c.getId())
            .category(c.getCategory().name()) // enum → String
            .build();
    }
}
🎮 Analogía
🎮 League of Legends — Interfaz del juego El modelo interno del campeón tiene 50 atributos internos. Pero la interfaz que el jugador ve (barra de vida, daño, habilidades) solo muestra los necesarios. El DTO es esa UI: filtra y presenta solo lo relevante para el frontend.
📚 Definición de Clase
DTO (Data Transfer Object) — Clase simple que solo transporta datos entre capas. No tiene lógica de negocio. El método estático fromDomain() es un factory method: crea instancias a partir de otro objeto.

🔧 BeanConfiguration.java Config
¿Qué hace?

El "pegamento" hexagonal. Crea los beans manualmente: primero el repositorio, luego lo inyecta en el servicio. Gracias a esta clase, el dominio NO necesita anotaciones de Spring.

@Configuration
public class BeanConfiguration {

    @Bean
    public CourseRepositoryPort courseRepositoryPort() {
        return new InMemoryCourseRepository();
    }

    @Bean
    public CourseUseCase courseUseCase(CourseRepositoryPort repo) {
        return new CourseService(repo);
    }
}
🎮 Analogía
⚗️ Resident Evil — Laboratorio de Mezcla de Hierbas Para mezclar una hierba verde + roja + azul necesitas hacer el proceso en orden. BeanConfiguration es ese laboratorio: primero crea el repositorio (hierba base), luego lo mezcla con el servicio para producir el objeto final curado que Spring inyecta donde se necesita.
🌍 CorsConfig.java Config
¿Qué hace?

Configura CORS (Cross-Origin Resource Sharing). Sin esto, el navegador bloquea las peticiones del frontend Angular (localhost:4200) al backend (localhost:8080) por estar en puertos diferentes.

registry.addMapping("/api/**")
    .allowedOrigins(
        "http://localhost:4200",
        "https://static-years-354369.framer.app"
    )
    .allowedMethods("GET", "POST", "PUT", "DELETE");
🎮 Analogía
🌫️ Silent Hill 2 — El Puente entre Mundos En Silent Hill, James cruza entre el mundo real y el mundo oxidado. Sin el puente (CORS), cada mundo está completamente aislado. CorsConfig es ese puente prohibido que habilitas tú: le dices al servidor quién puede cruzar y para qué.
